Lambda: What?
  • Lambda is a serverless compute service that lets you run code without provisioning or managing servers
  • It is a virtual function, runs for short period of time and runs on demand
Lambda: Why?
  • No servers to manage
  • Continuous scaling(auto scaling)
  • Consistent performance at any scale
  • Integrated with different services(eg Dynamodb, SQS, S3)
  • Integrated with many programming languages(eg Java, Go, PowerShell, Node. js, C#, Python, and Ruby)
Limitations
  • Function Memory allocation: 128MB-10GB
  • Maximum execution time 900sec(15 mins)
  • Function environment variables(4kb)
  • /tmp directory storage(512 MB)
  • Concurrent execution 1000(Can be increased up to Hundreds of thousands)
  • Function deployment size(compressed .zip): 50MB
  • Size of uncompressed deployment(code + dependencies): 250MB
Pricing
  • Pay per request and compute time
  • Free tier of 1,000,000 AWS Lambda requests and 400,000 GBs of compute time
